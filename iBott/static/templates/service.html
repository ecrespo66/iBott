¿Cómo funciona la Robot Console?

Es una aplicación web que se encarga de gestionar tus automatismos de una forma sencilla. Pensada para que pueda ser utilizada por cualquier usuario de negocio, sin necesidad de tener ningún background técnico.
Cuenta con un sistema único de carpetas en donde se agrupan todos los automatismos, puedes invitar a los miembros de tu equipo a participar en tu carpeta o proyecto y asignar diferentes permisos y roles.
Está compuesta por 3 apartados: Robots, Machines y Schedules con los se controla todo el ecosistema de robotización. Cuenta además con una página en donde se recogen las métricas principales de tus procesos. Regístrate en la consola, o contacta con nuestro equipo de robotización y te ayudaremos en todo el proceso.

Folders

Una vez creada nuestra cuenta en iBott Robot Console, se nos creará una carpeta que contiene todo lo relativo a nuestro proyecto de automatización. Dentro de esta carpeta, tendremos un usuario Robot asociado con un token de autorización con el que realizar el intercambio de datos entre la consola y las distintas máquinas configuradas para nuestro proyecto.
Si navegamos a nuestra carpeta, podemos ver una lista con los distintos grupos de usuarios creados con los diferentes permisos asociados a cada grupo.
Por defecto se creará el grupo admin en el que estareis tu y el robot como usuarios asociados. Aquí podrás añadir nuevos grupos, asignarles permisos e invitar a otros usuarios a trabajar en tu proyecto.

Una vez configurados los grupos y permisos, tendremos que configurar la conexión con github para poder ejecutar los procesos. Para ellos tendremos que crearnos una cuenta en github.com, navegar al apartado Settings -> Developer Settings -> personal access tokens. Generaremos un nuevo token con permisos de lectura y clonado de repositorios y copiaremos el token de acceso que tendremos que pegar en el apartado Settings de nuestro proyecto, asi como nuestro usuario de github.

Robots

Aquí encontraremos un listado con todos los procesos creados. Podrás ordenar y filtrar la lista de procesos, navegar a los procesos ya existentes o crear uno nuevo en el botón add robot + de la esquina superior derecha.

Una vez entremos en un proceso encontraremos los siguientes campos con la información necesaria para la ejecución de tus automatismos:

  1. Nombre del proceso. Se trata de un identificador único para cada automatismo, lo que significa que no podrás tener más de un proceso con el mismo nombre.
  2. Tags que identifican el proceso ej: Aplicaciones que utiliza.
  3. Url del repositorio en donde se encuentra alojado el robot. Sino conoces Github y la tecnología git, es hora de que aprendas un poco sobre esta impresionante tecnología aqui: Tecnología Git
  4. Formulario de entrada. Este formulario permite enviar información actualizada al robot de una forma rápida y sencilla. Puedes seleccionar fechas de un calendario, subir archivos para que el robot los procese, escribir números o texto, o incluso seleccionar elementos de un desplegable. No olvides definir la información por defecto a la hora de crear el formulario, pues esta será consumida por el robot cuando se lancen los procesos automáticamente.
  5. Máquinas. En este desplegable, seleccionaremos el ordenador o máquina en donde se ejecutará el proceso.
  6. Ramas (Branches). Aquí seleccionaremos la rama dentro del repositorio git, la cual queremos ejecutar, esto es muy util cuando tenemos dos o más versiones de un mismo proceso. Sino sabes que son las ramas, puedes consultar el siguiente articulo: Tecnología Git
  7. Documentación del proceso. Se trata de un archivo con la extensión .md que se encuentra alojado en el repositorio con la el nombre readme.md aquí encontraremos toda la información relativa al automatismo con el que vamos a trabajar.

Executions

En el apartado ejecuciones, encontraremos un listado con las ejecuciones de todos nuestros procesos. Puedes ordenarla y filtrarla con cualquiera de los campos visibles de la página.

Aquí encontraremos un resumen con toda la información necesaria para ver de un vistazo el estado de las ejecuciones:

  1. Identificador único de la ejecución
  2. Estado de la ejecución (OK, Fail, Pending o Working)
  3. Nombre del proceso ejecutado
  4. Fecha de inicio de la ejecución
  5. Fin del fin de la ejecución.

Una vez entramos en la ejecución, tendremos un listado de todos los logs que el proceso ha dejado para dicha ejecución donde se recoge la siguiente información:

  1. El tipo de log (Log, Trace, Info, System o Business Exception).
  2. El mensaje que contiene el log.
  3. La fecha en la que se registra el log.

Queues

Una Queue o cola de trabajo, es una tabla en la cual se almacena un conjunto de Items. Cada item contiene una con la información necesaria para cada transacción que el proceso tiene que realizar. Una vez dentro a la robot console, encontraremos dentro del apartado Queues un listado con todas las colas disponibles para tus procesos en donde encontraremos siguiente información:

  1. El nombre de la cola
  2. los robots que pueden publicar o consumir los datos de dicha cola
  3. El identificador único de la cola
  4. El número de ítems que contiene la cola
  5. El estado en el que se encuentra la cola, que puede ser (OK, Fail, Pending o Working).

Una vez dentro de la cola, encontraremos una lista con todos los items que pertenecen a dicha cola, en donde se recoge la siguiente información:

  1. Identificador del item o Id
  2. El estado en el que se encuentra (Pending, Working, Ok o Fail)
  3. La fecha de creación del item
  4. La fecha de resolución del item en el caso de que haya sido procesado

Una vez accedemos a un item, encontraremos una tabla dinámica de NxN celdas en la que se almacena toda la información a consumir por el robot.

Assets

Los assets se tratan de variables globales que pueden ser consumidas por uno o más procesos. Estas variables se alojan en la consola para que puedan ser modificadas de forma rápida y sencilla por el equipo no técnico, una única vez sin necesidad de cambiarlas proceso por proceso.
Aquí se almacena información relativa a credenciales de acceso, cuentas de correo para reportes, urls de acceso o cualquier otra información que sea consumida por los automatismos. Cuando accedemos al apartado Assets, encontraremos la siguiente información:

  1. El identificador de Asset o Id
  2. El nombre del asset
  3. El tipo de información que almacena, que podrá ser de tipo Credencial, Texto o Numérica

Machines

En el apartado máquinas, encontraremos el listado de máquinas disponibles para ejecutar los automatismos. En este apartado encontraremos la siguiente información:

  1. Estado de la máquina.Este podrá ser Closed, cuando esta se encuentre desconectada, Free cuando esta se encuentre disponible para ejecutar un proceso, o Busy cuando se encuentre ejecutando un automatismo.
  2. Nombre de la máquina. Para poder identificarla fácilmente.
  3. Id de la máquina y clave de licencia de la máquina. Ambos valores nos permiten realizar la conexión entre la consola y las distintas máquinas configuradas.
  4. Dirección IP y puerto de conexión en donde la consola enviará la información para realizar la conexión.

Schedules

En este apartado encontraremos tanto un programador de ejecuciones como un calendario en donde se registran todas las ejecuciones programadas de tus procesos.
En el primero, tenemos una interfaz gráfica que se encarga de generar tareas de tipo cron, donde tendrás que seleccionar el año, los meses, los días, las horas y los minutos en los que se ejecutarán tus automatismos, pudiendo incluso generar lógicas complejas o periodos de tiempo.

En el segundo tenemos un calendario con tres vistas, mensual, semanal y diaria, asi como una tabla donde se listan todas las tareas guardadas para tus procesos.

Charts

Por último, encontraremos una página donde se recogen las métricas de tus procesos. En la parte superior de la página, encontraremos 4 desplegables donde podremos filtrar la información por robot, fecha, cola y status.
A continuation, tenemos las métricas totales de la aplicación:

  1. Número de procesos.
  2. Número de ejecuciones.
  3. Número de colas.
  4. Número de items totales.

En el siguiente apartado encontramos un gráfico de tipo tarta en donde se recoge el total de ejecuciones por estado y un gráfico de barras apilado en donde podemos ver el número de ejecuciones y su estado por cada robot.
Por último, tenemos la información sobre las colas y los items, en donde al igual que en el apartado anterior, se recoge el total en un gráfico de tarta y la información desglosada por colas en un gráfico de barras apilado.
